## 2026-07-19：doctor 中的中文项目路径显示乱码（已解决）

- 症状：PowerShell 输出的仓库路径 `D:\codex\项目\Project_01_finance-tracker` 正常，但 Python 子进程输出的 `sys.executable`、`sys.prefix` 和 pip 路径显示为 `D:\codex\��Ŀ\Project_01_finance-tracker`。
- 分类：编码 / 环境显示。
- 影响：doctor 实际退出码为 0，测试也通过；但输出不满足项目 UTF-8 显示要求，可能掩盖其他非 ASCII 路径问题。
- 最小复现：从 PowerShell 调用 `.\.venv-phase1\Scripts\python.exe -c "import sys; print(sys.executable)"`，并将输出与 PowerShell `Resolve-Path` 结果比较；修复前稳定失败。
- 根因：当前 Windows 活动代码页为 936；Python 3.14 子进程报告 `stdout=gbk`、首选编码 `cp936`、UTF-8 模式关闭，而 PowerShell 7 的 `$OutputEncoding` 为 UTF-8。`doctor.ps1` 未像 `test.ps1` 一样显式设置 Python UTF-8 输出，导致 GBK 字节被 PowerShell 按 UTF-8 解码。
- 验证：临时设置 `PYTHONIOENCODING=utf-8`，或仅给 Python 增加 `-X utf8` 后，最小复现输出正确路径并与 `Resolve-Path` 完全一致。
- 建议修复：在所有会捕获或解析 Python 文本输出的 PowerShell 脚本中统一显式设置 `PYTHONUTF8=1` 和 `PYTHONIOENCODING=utf-8`，并增加包含中文字符的回归检查。
- 长期方案：由仓库入口脚本自行固定 Python UTF-8 模式，不依赖激活脚本、PowerShell Profile、`chcp 65001` 或 Windows 全局区域设置；`doctor.ps1` 增加使用 ASCII 转义生成中文哨兵文本的子进程检查，确保即使项目迁移到纯 ASCII 路径也能捕获编码回归；`AGENTS.md` 的编码规则补充为所有 PowerShell→Python 文本边界的强制要求。
- 实施：`scripts/doctor.ps1` 和 `scripts/bootstrap.ps1` 已自行设置 `PYTHONUTF8=1` 与 `PYTHONIOENCODING=utf-8`；`doctor.ps1` 已增加由 ASCII 转义生成“项目”的 Unicode 哨兵检查；`AGENTS.md` 已加入 PowerShell→Python 文本边界规则。
- 验证：doctor 中的仓库、Python、pip 路径均完整显示中文；Unicode 哨兵通过；`scripts/doctor.ps1` 退出码为 0；`scripts/test.ps1` 完成 69 项测试并全部通过。
