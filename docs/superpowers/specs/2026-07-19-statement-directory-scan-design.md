# Issue #17：账单目录扫描与增强导入预览设计

## 目标

从项目根目录的 `银行流水/` 递归发现账单文件，让用户选择尚未导入的文件进入现有批量预览与确认流程，同时将文件名约定转换为可筛选账户。

## 扫描边界

- 扫描根目录固定为 `银行流水/`，递归收集 `.pdf` 与 `.csv` 文件。
- 扫描只读；不移动、删除、上传、解析或写入原始文件。
- 每个发现文件计算 SHA-256，并与 `source_files.sha256` 比对，标记已导入文件。
- 手动选择目录外文件的现有路径继续可用。

## 账户归属

文件名规则在扫描结果与确认写入中一致：

- `.pdf`：`ME`；
- `-czj.csv`：`ME`；
- `-cr.csv`：`WIFE`；
- 其他 CSV：`needs_account_selection`，不自动进入确认。

账户归属在预览中可见，并覆盖解析器的来源默认账户，使交易、筛选与后续报表按 `ME`、`WIFE` 区分。

## 用户流程

1. 用户在导入页面请求扫描。
2. 服务返回相对路径、文件类型、账户归属、已导入状态与可选状态；默认不选中已导入和未知归属 CSV。
3. 用户选择文件，服务读取选中文件并调用既有 `preview_many()`。
4. 预览显示来源路径、账户归属、规范商户、分类、规则依据和审计状态。
5. 用户确认后沿用既有原子批量写入；扫描本身和预览均不写入账本。

## 失败处理与验证

- 不可读、格式不支持或解析失败的文件作为单文件错误返回，其他文件继续预览。
- 相同 SHA-256 的目录副本显示已导入，不会重复写入。
- 测试只使用 `.tmp/` 中的合成 PDF/CSV 文件，覆盖递归发现、账户归属、未知 CSV、重复源和目录外手动导入兼容。
- 本 Issue 不实现自动导入、原始文件管理、复核界面或报表页面。
