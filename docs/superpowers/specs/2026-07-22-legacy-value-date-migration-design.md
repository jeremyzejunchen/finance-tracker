# 旧账本 value_date 兼容迁移设计

## 目标

让缺少当前导入字段的历史 SQLite 账本在启动时安全升级，使 CSV 导入可以写入账本。

## 决策

采用受限的完整列迁移：仅在列缺失时补齐当前确认写入路径使用且具有安全默认值的字段：`transactions` 的 8 个字段（`value_date`、原始商户、交易类型、来源元数据和两个布尔标记），以及 `reconciliations` 的退款配对原因和状态。迁移前沿用现有哈希校验备份机制；已有记录不改写，后续导入写入解析出的字段值。

不实现任意列或任意表的通用结构修复器；迁移清单严格限定为当前确认写入路径需要、且 SQLite 可通过默认值安全添加的列。

## 验证

- 合成旧数据库模拟正式历史交易或对账列集合时，`Database.initialize()` 创建备份并补齐全部缺失列。
- 补齐后通过 `FinanceService.confirm_many()` 成功写入合成 CSV 交易。
- 项目完整测试通过，并使用应用内浏览器运行一次临时数据库的导入与商户复核回归。
