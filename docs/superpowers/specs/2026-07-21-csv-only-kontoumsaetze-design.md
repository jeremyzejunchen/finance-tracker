# CSV-only Kontoumsaetze 数据源迁移设计

## 目标

将应用迁移为仅接受 CSV 账单，删除 PDF 导入能力，并让 `Kontoumsaetze*-czj.csv` 成为 ME 银行流水的正式来源。现有账本中的 PDF 来源数据会在本地备份后彻底移除，使 CSV 成为唯一统计来源。

## 数据源与账户归属

- 账单目录只发现 `.csv` 文件；PDF 不再扫描、上传、预览或导入。
- CSV 内容决定解析器，文件名后缀决定账户：`*-czj.csv` 强制为 `ME`，`*-cr.csv` 强制为 `WIFE`，其他后缀需要人工选择。
- `Kontoumsaetze` 格式目前只出现在 `*-czj.csv`，由专用导入器识别；其他 CSV 保持各自的内容识别与解析路径。

## Kontoumsaetze 解析与清洗

导入器只接受 UTF-8、分号分隔的 `Kontoumsaetze*-czj.csv`。它跳过账单元数据区，并从交易表列标题开始读取交易记录。

交易提取使用记账日、价值日、金额、币种、交易类型、收付款方、用途与参考字段。IBAN、BIC、客户参考、授权参考及其他账户标识不进入页面、日志、测试夹具或 Git。

商户清洗优先使用收付款方；为空或为通用标签时，从用途提取可识别商户。无法可靠识别的交易保留为待复核，不能静默分类。

缺失交易表标题、不可解析的日期/金额/币种会使整份文件在预览阶段失败而不写入账本。单行缺失核心字段时保留可审计 warning。

## PDF 来源移除

启动 CSV-only 版本时，若账本存在 `deutsche_bank_pdf` 来源：

1. 先在项目内创建带时间戳的数据库备份。
2. 在单个 SQLite 事务中删除 PDF 来源交易，以及仅指向这些交易的对账、审计、导入批次和来源文件记录。
3. 写入不含交易细节的迁移审计摘要和 schema 版本。

迁移必须幂等。事务失败时数据删除回滚，备份保留。其他 CSV 来源和交易保持不变。用户磁盘中的 PDF 原件绝不删除。

PDF 解析器、PDF 文件检测、PDF 专属 UI 文案、测试与文档从代码库删除。

## 验证与隐私

所有自动化测试及浏览器测试仅使用匿名合成 CSV 与临时数据库，覆盖格式识别、账户归属、字段提取、商户清洗、错误输入、PDF 拒绝、PDF 账本迁移与幂等、其他 CSV 来源保留、目录扫描和确认导入。

完成后执行一次真实本地数据库的只读汇总验收：PDF 来源交易数必须为零；只报告 CSV 来源交易数、待复核数和商户覆盖率等汇总，不输出交易、账户、文件名或金额明细。

## 非目标

- 不删除用户磁盘中的 PDF 原件。
- 不迁移或重写旧 Issue #18 的商户复核设计。
- 不将真实账单或任何敏感元数据加入 Git、测试、日志或截图。
