# 将运行数据保存在项目目录内

Finance Tracker 的正式 SQLite 账本、私密配置和其他持久运行数据统一放在项目内的 `data/`，处理与测试产生的临时文件放在 `.tmp/`，用户主动导出的报告或备份放在 `exports/`。这些目录必须由 `.gitignore` 保护，只提交虚构的配置示例；这一选择提高了项目的自包含性，代价是复制、清理或删除项目目录时必须同时谨慎对待其中的真实财务数据。

从旧 `%LOCALAPPDATA%` 位置迁移时，应用先在项目内创建时间戳备份，再复制账本和私密配置到 `data/`；只有备份与目标副本都已逐字节校验一致时，才删除旧原件。复制、校验或备份失败时必须保留旧原件。
