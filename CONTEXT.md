# Finance Tracker

Finance Tracker 将个人财务来源整理为可审计、可复核且可统计的本地交易记录。

## Language

**本地记账闭环**:
从原始银行与支付平台文件导入开始，经过数据清理、审计、复核和分类，最终形成可信网页统计的完整本地工作流。
_Avoid_: 仅完成 Phase 1、仅生成报表

**商户规则基线**:
`legacy/categories.md` 中商户关键字与分类之间的对应关系，是衡量数据清理覆盖率和分类结果的历史参考；它不是逐笔交易真值。
_Avoid_: 旧版逐笔结果、绝对字符串一致

**商户覆盖率**:
参与统计的交易中，清理后商户能够命中商户规则基线并得到对应分类的比例；内部转账、换汇和失败交易不计入分母。完成门槛为至少 95%，其余交易必须显式进入待复核。
_Avoid_: 所有交易命中率、静默兜底分类率

**报表**:
从账本派生的只读筛选、汇总、趋势、图表和交易明细视图；报表不保存分类规则、标签或其他账本状态。
_Avoid_: 设置页、独立 HTML 数据副本

**退款**:
与先前支出配对的正向交易；账本保留退款与原支出两条记录，报表用退款冲减原支出分类，而不把退款计作收入。
_Avoid_: 退款收入、排除退款配对

**非消费资金流**:
内部转账、换汇和投资入金或出金等不代表日常收入或消费的资金移动；它们不计入收入、支出和净额，但在报表中单独汇总并可查看明细。
_Avoid_: 普通收支、隐藏交易

**外部换汇资金流**:
从尚未纳入账本的人民币账户换汇后转入已记录账户的资金；由于来源账户不在账本中，它不属于内部转账，也不能计作收入、支出或净额，只作为可审计的非消费资金流保留。
_Avoid_: 收入、内部转账、隐藏交易

**人工覆盖**:
用户只针对一笔交易确认的商户或分类结果；它优先于自动清理和商户规则，重新导入或规则更新不得静默覆盖。
_Avoid_: 临时规则、自动分类

**商户规则**:
用户确认的、可复用于现有同商户交易和未来导入的商户与分类对应关系；保存前必须明确展示影响范围。
_Avoid_: 单笔修正、静默批量修改

**商户复核组**:
由同一清理后商户关联的待复核交易集合；用户可以对整组确认商户规则，并对组内特殊交易保留人工覆盖。
_Avoid_: 分类批次、不可拆分批量修改

**规范商户**:
用户可识别且用于复核、分类和报表汇总的统一商户身份；多个原始商户写法可以对应同一个规范商户。
_Avoid_: 原始说明、匹配关键字

**商户别名**:
经用户确认、用于把精确名称或包含关键字对应到规范商户的匹配方式；应用前必须能够预览影响范围。
_Avoid_: 人工覆盖、任意正则表达式

**账本**:
用户确认写入的本地交易记录及其来源、复核、分类和对账关系，是报表的唯一事实来源。
_Avoid_: 导入预览、旧版 JSON 缓存、报表数据副本

**导入预览**:
账单写入账本前展示的非持久候选结果，包括规范商户、分类、命中规则、审计发现和复核原因；只有用户确认后才写入账本。
_Avoid_: 已导入交易、自动写入

**账本质量**:
以整个账本的商户覆盖率衡量的数据清理状态；低于 95% 时显示质量警告并产生待复核任务，但不阻止单个导入批次写入。
_Avoid_: 单批次通过率、导入阻断条件

**待分类交易**:
已经确认写入账本、但尚未获得最终分类的真实现金流；它计入收支总额与趋势，并在分类统计中单列为“待分类”。
_Avoid_: 排除交易、未导入交易

**账单目录**:
项目内保存原始银行与支付平台文件、供应用只读扫描并生成导入预览的本地目录；扫描本身不会写入账本或修改源文件。
_Avoid_: 自动导入目录、临时上传目录
